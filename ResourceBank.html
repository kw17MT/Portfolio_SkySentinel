DDSファイル、テクスチャのリソース再利用

今回私が取り組んだことはモデル描画時に使用するテクスチャファイルの再利用ができるようにエンジンを改造しました。学校ゲームエンジン内にはモデルリソースの再利用がすでに実装されてありますので、それを読み解きながら行いました。
実装を行った経緯としては、私はこれまで学校ゲームエンジン内の処理を追いかけることを避けていました。そうしたこともあり、いざ他のエンジンを取り扱うこととなると対応が遅れてしまいました。今後、他エンジンを取り扱う際に再び後れを取らないように上記のことに挑戦しました。また、エンジン内部の処理や流れの一部分を知っておくだけでも今後の制作活動に役立ち、他エンジンを触る際には参考になると考えたからです。

DDSファイル、テクスチャの再利用を選んだ理由

私が現在制作しているゲームはモデルによるメモリ圧迫に問題がありました。改善前であるとタイトル画面で10GB、インゲームで20GB前後と環境によっては実行ファイルすら開くことができない状態でした。問題はモデルがパーツ毎に細かく小分けにされている状態であるために、それに対応するテクスチャを大量に生成する必要があったためです。

リソース再利用の流れ

    使用するクラスの概要
    Model…モデル描画に必要なファイルパス、シェーダーのエントリーポイント、カラーバッファをまとめて持つクラス。
    TkmFile…学校エンジンでのみ使用できるモデルファイル。各種DDSファイルと頂点バッファを持つクラス。3dsMaxから出力するファイルを読み取る。
    Texture…DDSファイルを読み込むクラス。
    Meshparts…TkmFileとテクスチャをもとにモデルを描画するクラス。
    Material…DDSファイルからテクスチャを生成するクラス。
    ResourceBank…一度読み込んだモデル、DDSファイル、テクスチャをファイルパスをキーに保存しているクラス。
  1. Model → TkmFile → ResourceBank
    Modelクラスに設定したファイルパスを使用し、TkmFileにモデル情報を設定します。この時、ResourceBankを参照して同一ファイルパスが保存されていたら、その値のポインターをTkmFileに与え、重複した生成処理を行わないようにします。 同一ファイルパスが存在しなければ、TkmFile内でファイルパスとモデル情報を保存し、ResourceBankにもパスをキーとしてモデル情報を保存します。これで次から同じファイルパスを設定したオブジェクトは保存済みのモデル情報を使用できます。（メモリ圧迫回避）
  2. TkmFile → Texture → ResourceBank
    次に、TkmFileから読み込んだ使用されているDDSファイルパスを用いて、既にResourceBankに同一ファイルパスとDDSファイルが保存されていないか確認する。モデル情報と同様に、あったらポインターを取得、なければファイルパスとDDSファイルを保存する。
  3. TkmFile → MeshParts
    モデル情報とDDSファイル情報を保持したTkmFileをMeshpartsクラスに渡します。
  4. Meshparts → Material → ResourceBank
    MeshpartsクラスからMaterialクラスにDDSファイルパスを渡し、DDSファイルパスを用いてテクスチャを生成します。この時もResourceBankに同一ファイルパスから既にテクスチャを生成していないか問合せ、あればポインターを取得、なければテクスチャを生成しファイルパスと共にResourceBankに保存します。
  5. Material → Meshparts
    最後にMeshpartsでモデル情報からメッシュを生成し、Materialを適用し描画します。

簡単なイメージ

#　結果 最終的に以下にメモリ使用量を抑えることができました。
タイトル画面でのメモリ使用: 10GB → 1.7GB
インゲームでのメモリ使用　: 20GB → 2.9~3.0GB
ロード時間に関しても全くストレスのない速度になりました。
